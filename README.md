# 점자 플레이트 STL 생성기 (Braille Plate STL Generator)

한글, 영문, 숫자 텍스트를 입력하면 3D 프린터로 출력 가능한 **점자 플레이트 STL** 파일을 생성하는 Python GUI 앱입니다.

![원본 demo](braille1.png)

## ⚡ 핵심 강점

- 🎯 **국립국어원 한국 점자 규정 PDF 직접 준수** — 2017 + 2020 개정 해설서 PDF를 NABU ASCII Braille 디코딩으로 직접 검증. **단독 약자 11자 + VF 약자 14종** (을 약자 포함) + **다음절 약어 7종** + **제15·16항** 특례 (성/정/청 특별 영 약자, 셩/졍/쳥 풀어쓰기 반대 룰, 겹받침 첫 자음 약자 매칭) + **컨텍스트 의존 부호** (한글 본문 vs UEB 로마자) 모두 구현. **128 케이스 round-trip 테스트** 통과 (`tests/smoke_braille.py`).
- 🖨 **FDM 프린팅 친화 기하** — **Dome 점자** (원기둥 앵커 + 평평 꼭대기) 로 점 탈락·까끌함 방지. **필렛 플레이트** (윗면·옆면 라운드 / 바닥면 평면 = 배드 접착).
- 🧑‍🔧 **설치자 배려** — 뒷면에 방향 표시 **음각 삼각형** + **원본 텍스트 (자동 X 미러로 정자 가독)**. 점자 모르는 사람도 어느 쪽이 위인지 / 무슨 라벨인지 즉시 식별.
- 🔍 **그냥 출력하면 정상** — Y축 자동 미러로 슬라이서 Y-up 좌표계와 호환. 슬라이서에서 별도 회전·반전 불필요.
- 🖱 **GUI + 4 프리셋** — Tkinter UI · 입력 즉시 Unicode 점자 미리보기 · trimesh 3D 뷰어. **A 안정형 / B 박형 / C 사이니지 / D 카드형** 한 클릭.
- 📜 **GPL-3.0 + 정직한 개발 일지** — 거쳐온 버그 5건 README 에 그대로 기록 (반성문 섹션). 같은 실수 재발 방지.

## 주요 기능

- **한글 · 영문 · 숫자** 입력 → 자동 점자 변환 (국립국어원 한국 점자 규정 PDF 직접 준수)
  - 2017 한국 점자 규정 **단독 약자 11자** (가·나·다·마·바·사·자·카·타·파·하)
  - **VF 약자 14종** (억·언·얼·연·열·영·옥·온·옹·운·울·은·**을**·인) 자동 적용
  - **다음절 약어 7종** (그래서·그러나·그러면·그러므로·그런데·그리고·그리하여)
  - **제15·16항 특례** — 성/정/청 영 약자 적용 / 셩/졍/쳥 풀어쓰기 / 겹받침 첫 자음 약자 매칭
  - **컨텍스트 의존 문장 부호** — 한글 본문은 한국 점자 규정 (제44~67항), 로마자 사이는 통일 영어 점자 (UEB)
- **Dome 점자** (기본) — 원기둥 앵커 + 반구 캡, FDM 프린터에서 탈락 없이 잘 붙는 기하
- **필렛된 백플레이트** — 윗면/옆 모서리는 라운드, 바닥면은 평면 유지 (프린터 배드 접착)
- **뒷면 음각 삼각형** — 설치자가 플레이트 위/아래 방향을 즉시 구별 (Boolean 차집합)
- **4가지 프리셋** (A 안정형 / B 박형 / C 사이니지 / D 카드형) — 용도별 파라미터 일괄 설정
- **프린팅 지지대** 토글 (얇은 플레이트가 프린팅 중 휘지 않도록)
- **Tkinter GUI** + Unicode 점자 (U+2800 ~ U+28FF) 실시간 미리보기
- **trimesh 3D 뷰어** 로 저장 전 STL 미리보기 (별도 프로세스, GUI 안 멈춤)
- **바이너리 STL** 출력 (numpy-stl)

## 설치

Python 3.10+ 권장.

```bash
pip install -r requirements.txt
```

`requirements.txt` 내용:
- `numpy`, `numpy-stl` — 메시 연산 & STL 직렬화
- `trimesh[easy]` + `pyglet<2` — 3D 미리보기 뷰어
  - (trimesh 4.x 내장 뷰어가 아직 pyglet 2.x 미지원이라 1.5 계열로 핀)
- `manifold3d` — 뒷면 음각 삼각형을 위한 Boolean 차집합 연산

## 실행

```bash
python3 app.py
```

1. 텍스트 입력란에 원하는 문구 입력 (Enter 로 줄바꿈)
2. **프리셋 A/B/C** 버튼으로 용도별 파라미터 원클릭 적용 (또는 직접 조정: 두께 · 필렛 · 점 반경 · 음각 등)
3. **3D 미리보기** 로 모양 확인 → **STL 파일로 저장**

## 개발 배경

이 프로젝트는 [**benjaminaigner/braillegenerator**](https://github.com/benjaminaigner/braillegenerator) 의 `braille.jscad` (OpenJSCAD) 파일을 참조하여 **한글 점자 지원을 추가**하고 **Python + Tkinter GUI 로 포팅**한 파생 저작물입니다.

### 원본에서 계승한 것

- 점자 셀의 기하 상수 (점 지름 1.44 mm, 셀 내 점 간격 2.5 mm, 문자 간격 6 mm, 줄 간격 10.8 mm, 플레이트 두께 2 mm)
- 2 mm 백플레이트 + 인쇄 지지대 구조
- 문자 → 점자 비트맵 매핑의 기본 아이디어

### 새로 구현한 것

#### 1. 한국어 점자 (국립국어원 2017 한국 점자 규정)

`braille_data.py`:

- **초성 19자** (`ㄱ~ㅎ`, 쌍자음 포함) — 쌍자음은 `ㅅ`(dot 6) 프리픽스 + 평자음 2셀
- **중성 21자** — `ㅘ`, `ㅙ`, `ㅝ`, `ㅞ`, `ㅟ`, `ㅢ`, `ㅒ` 등 복모음은 2셀
- **종성 28자** — 겹받침 (`ㄺ`, `ㄻ`, `ㄼ`, `ㄽ`, `ㄾ`, `ㄿ`, `ㅀ`, `ㅄ`, `ㄳ`, `ㄵ`, `ㄶ`) 모두 2셀 분해
- Hangul Syllables (U+AC00 ~ U+D7A3) 자동 **초성+중성+종성** 분해
- `ㅇ` 초성 생략 규칙 적용
- **단독 약자 11자** (제29항) — 자음+ㅏ(받침X) 음절을 단일 셀로. 카 → ⠋, 가 → ⠫, 사 → ⠇ 등
- **VF 약자 13종** (제29항) — (모음, 종성) 조합 단일 셀 축약. 예: 신(ㅅ+ㅣ+ㄴ) → ⠠⠟ (ㅅ초성 + 인약자), 분(ㅂ+ㅜ+ㄴ) → ⠘⠛ (ㅂ초성 + 운약자)

검증 레퍼런스: [t.hi098123.com/braille](https://t.hi098123.com/braille) 의 변환 결과와 일치 (신분증 → ⠠⠟⠘⠛⠨⠪⠶, 카드입구 → ⠋⠊⠪⠕⠃⠈⠍)

#### 2. 영문 / 숫자

- Grade-1 영문 점자 (a-z)
- 숫자 prefix `⠼` (dots 3-4-5-6) + letter sign `⠰` (dots 5-6) 전환
- 대문자 prefix `⠠` (dot 6)
- 기본 문장부호 (`. , ? ! ; : ' - ( ) "`)

#### 3. Python 메시 파이프라인 (`generator.py`)

- 순수 `numpy` 로 UV sphere · Dome(원기둥+반구) · axis-aligned box · 필렛 플레이트 메시 생성 (모두 watertight, 외향 normal)
- 점자 ↔ 플레이트 결합: **오버랩 union** (슬라이서 자동 병합)
- 뒷면 음각: **`trimesh + manifold3d` Boolean 차집합** — 필렛된 라운드 뒷면에서도 깨끗하게 파냄
- `numpy-stl` 로 바이너리 STL 직렬화

#### 4. 필렛 백플레이트 (사용자 요청으로 추가)

3D 프린팅에 친화적인 "위·옆은 둥글게, 바닥은 평면" 플레이트:

- **스켈레톤 사각형** `[r, W-r] × [r, H-r]` 을 기준으로 높이별 오프셋
- 수직 영역 `z ∈ [-t, -r]`: 외측 오프셋 `d = r` 고정 → **수직 모서리 4개** 라운드
- 상단 필렛 `z ∈ [-r, 0]`: `d(z) = √(r² - (z+r)²)` 로 수축 → **윗면 모서리 4개** 1/4 원호
- 바닥면 `z = -t`: 라운드 사각형이지만 **측벽과는 90° 샤프** 유지 → 프린터 배드 접착면 보장
- `r` 자동 클램프: `min(r, W/2, H/2, t) − eps`
- 링 기반 스윕 + 팬 트라이앵글레이션 → `trimesh.is_watertight == True` 보장

#### 5. Tkinter GUI (`app.py`)

- 텍스트 입력 + 옵션 위젯 (백플레이트, 지지대, 두께, 필렛 반경, 점 스타일/반경/박힘, 음각 ON·변·깊이)
- **프리셋 A/B/C 버튼** — 클릭 한 번으로 플레이트·필렛·점·음각 일괄 세팅
- 입력 즉시 Unicode 점자 미리보기 갱신
- 플레이트 최종 치수 실시간 표시

#### 6. 3D 뷰어 (`preview_stl.py`)

- **서브프로세스**로 실행 → Tkinter 메인 루프 블로킹 없음 (특히 macOS)
- `trimesh.Scene.show()` + pyglet 뷰어
- 임시 STL 파일 생성 → 프로세스에 경로 전달 → 앱 종료 시 `atexit` 로 정리

#### 7. Dome 점자 (탈락 방지 + 평평 꼭대기)

실제 3D 프린트에서 **구(sphere) 점자가 탈락**하는 문제를 겪고, **Dome** (원기둥 앵커 + 반구 캡) 기하로 기본값을 전환했습니다.

- 원기둥이 플레이트 안으로 `embed_depth` (기본 0.15 mm) 박혀 **앵커** 역할 → 층간 seam 분산
- 반구 캡의 반경 = 가시 높이 (ADA 점자 가이드 0.6 ~ 0.9 mm)
- 접촉 면적 π × r² ≈ **2.0 mm²** (구 접선 접촉 대비 +23%)
- **꼭대기 평평 깎기** (`flat_top_depth`, 기본 0.05 mm, on/off 토글) — 반구 꼭지를 살짝 절단해 FDM 노즐이 '점'이 아닌 **작은 면 (지름 ≈ 0.6 mm)** 에서 종료하도록 하여 뾰족 꼬리/블롭 제거
  - 기하 수식: 평면 캡 반경 = √(2 R d − d²), R = 점 반경, d = 깎기 깊이
  - GUI 체크박스로 언제든 OFF 가능 (레거시 뾰족 꼭지)
- 레거시 `sphere` 스타일도 옵션으로 유지

부수적으로 기존 `uv_sphere` 의 winding 버그 (삼각형이 안쪽을 향함, volume 음수) 도 함께 수정.

#### 8. 프리셋

용도별로 **플레이트 · 필렛 · Dot · 음각** 파라미터를 일괄 설정:

| 프리셋 | 용도 | 두께 | 필렛 | 점 반경 | 점 박힘 | 음각 (변 × 깊이) |
|---|---|---|---|---|---|---|
| **A 안정형** | 일반 사용 | 2.0 mm | 1.5 mm | 0.8 mm | 0.15 mm | 8.0 × 0.5 mm |
| **B 박형** | 벽·면 부착 | 1.2 mm | 0.6 mm | 0.75 mm | 0.20 mm | 6.0 × 0.3 mm |
| **C 사이니지** | 공공 표지 | 2.5 mm | 2.0 mm | 1.0 mm | 0.20 mm | 10.0 × 0.7 mm |
| **D 카드형** | 들고 다님 | 2.0 mm | 1.5 mm | 0.8 mm | 0.15 mm | 6.0 × 0.4 mm |

> 카드형 D 는 다른 프리셋과 동일 점자 크기지만 **여백을 5 mm 로 늘려** 손에 쥐기 좋은 카드 형태가 됩니다 (A/B 는 여백 2 mm, C 는 10 mm).

#### 9. Y축 미러 (3D 뷰어 · 프린터 호환)

내부 좌표계는 편집 편의상 Y가 아래로 증가(line 0 상단, line N 하단) 하지만, STL 출력 시 **Y-up 뷰어 및 3D 프린터 축 관습** 에 맞게 점자 배치를 Y축 기준으로 거울 반전:

- 라인 순서: `effective_line = (num_lines - 1) - line_idx`
- 셀 내 점 위치: `dy_flipped = 2 × DOT_SPACING - dy`
- 플레이트는 Y 대칭이라 영향 없음, 지지대도 `y = plate_h` 에 고정

결과: 뷰어에서 회전·줌해도 점자가 정상 순서로 읽히고, 프린트 후 손에 쥐어도 입력 텍스트 순서가 유지됨.

#### 10. 뒷면 음각 — 삼각형 (방향) + 원본 텍스트 (정자)

시각장애인 사용자가 아닌 **플레이트를 부착하는 사람** 을 위한 기능 — 플레이트 뒷면에 두 종류 음각:

**(1) 정삼각형** (설치 방향 표시):
- 위치: 둘 다 켜져 있으면 뒷면 **상단** (`cy = plate_h × 0.72`), 단독이면 중앙
- 방향: apex 가 **+Y 방향** (설치자가 뒷면을 볼 때 "위쪽")
- 자동 클램프: `size ≤ min(W·0.4, H·0.6)`, `depth ≤ thickness·0.4`

**(2) 원본 텍스트** (정자로 음각, 점자 모르는 설치자가 식별 가능):
- 위치: 둘 다 켜져 있으면 뒷면 **하단** (`cy = plate_h × 0.28`), 단독이면 중앙
- 폰트 자동 탐색: macOS `Apple SD Gothic Neo` / Linux `NanumGothic` / Windows `Malgun Gothic` (한글·영문·숫자 모두 지원)
- **X축 자동 미러** — 뒷면(-Z)에서 볼 때 정자로 읽히도록 폴리곤을 X 반전
- 폰트 크기 자동 축소: 폴리곤이 plate 안에 안 들어가면 비례 축소 (확대는 안함)
- `matplotlib TextPath` 로 글리프 → shapely (Multi)Polygon (even-odd rule via `symmetric_difference`) → `trimesh.creation.extrude_polygon` → Boolean 차집합

두 음각 모두 같은 `engraving_depth` 공유. 점자를 읽는 데는 전혀 영향 없음 (반대면).

- **[USER]** — 사용자가 실제 인쇄·확인한 ground truth (신분증 / 카드입구 / 영수증출구)
- **[REF]** — [t.hi098123.com/braille](https://t.hi098123.com/braille) 변환기와 cross-check
- **[RULE]** — 한국 점자 규정 제29항 (단독 약자 11자 + VF 약자 13종) 직접 도출

테스트 카테고리: 단독 약자 11개 standalone · 단독 약자 다음 음절 조합 · VF 약자 13개 (ㅇ초성) · VF 약자 with 다른 초성 (먹/먼/절/편/별/형/녹/손/동/출/근/민) · 일반 받침 음절 · 복모음 (ㅘ/ㅢ/ㅚ/ㅝ) · 다음절 문구.

## 🙇 반성문 — 한국 점자 매핑 시행착오

이 프로젝트의 한국 점자 부분은 **사용자가 실제로 인쇄·사용하며 발견한 버그를 수차례 수정**한 결과입니다. 첫 매핑을 AI 가 자기 기억에 의존해 적은 결과 작은 오류와 누락이 곳곳에 있었고, 사용자가 [t.hi098123.com/braille](https://t.hi098123.com/braille) 변환기와 cross-check 하며 잡아주셨습니다. 같은 실수 반복하지 않으려고 기록을 남깁니다.

### 1. 단독 약자 11자 누락 (`5bcddac`)
첫 매핑에 **가·나·다·마·바·사·자·카·타·파·하** 단독 약자 (한국 점자 규정 제29항) 가 통째로 빠졌었습니다. VF 약자 13종은 적었으면서 단독 약자 11자는 빼먹은 어이없는 누락. "**카드입구 → ⠋⠊⠪⠕⠃⠈⠍ 인데 왜 ⠋⠣⠊⠪⠕⠃⠈⠍ (8셀) 로 나오냐**" 라는 사용자 지적으로 발견.

### 2. 울 약자 점 패턴 오타 (`3f3faa8`)
`(ㅜ, ㄹ)` 약자 점 위치를 `[2, 3, 4, 6]` (⠮) 으로 잘못 적음. 표준은 `[1, 2, 3, 4, 6]` (⠯). "**얌마 영수증출구 잘못됐어 ⠻⠠⠍⠨⠪⠶⠰⠯⠈⠍ 이거여야돼**" — 출 (ㅊ+ㅜ+ㄹ) 케이스로 잡힘.

### 3. uv_sphere winding 잠복 버그 (`4e98257`)
점자 구 메시의 삼각형이 **안쪽 normal** (volume 음수) 이었는데 슬라이서들이 자동 수정해 줘서 출력엔 문제 없었음. Dome 추가하면서 우연히 발견. 메시 검증할 때 `is_watertight` 만 보지 말고 **부호 있는 volume** 도 확인해야 한다는 교훈.

### 4. on_save 의 fillet_radius 누락 (`c4812d3`)
`replace_all=True` 로 두 callsite 동시 변경한다고 적었는데 한쪽만 매칭됨. **3D 미리보기에는 필렛 적용 / STL 저장에는 미적용** 이라는 silent bug 가 여러 commit 동안 잠복.

### 5. 자체 테스트 기댓값 오타 다수
운전·역사·옥상·공장 검증 시 `⠶` 와 `⠬`, `⠊` 와 `⠣` 같은 점자 unicode 를 거꾸로 적어서 "코드가 잘못된 거 아닐까" 하고 추적했는데 알고 보니 **내 기댓값이 틀림**. 코드 멀쩡한데 self-doubt 한 부끄러움.

### 6. 은 약자 점 패턴 오타 (`250f863`)
`(ㅡ, ㄴ)` = 은 약자를 `[1, 3, 4, 5, 6]` (⠽) 로 적어왔는데 **국립국어원 한국 점자 규정 해설서 PDF** (`은빛 = z^o2` 예시) 를 NABU ASCII Braille 로 직접 디코딩하니 **`[1, 3, 5, 6]` (⠵)** 가 정답. `y` 와 `z` 의 한 끗 차이를 1년 가까이 못 잡았던 것. **t.hi098123 같은 부차 도구가 아니라 1차 출처 (PDF 본문) 를 보자**.

### 7. 을 약자 (14번째 VF) 통째 누락 (`250f863`)
한국 점자 규정 **제15항 약자는 14종** (억·언·얼·연·열·영·옥·온·옹·운·울·은·**을**·인) 인데 코드엔 13종만 있었음. 결과적으로 **'글, 글다, 늙다, 읊다, 가을'** 등이 풀어쓰기 (3 셀) 로 나와 길이가 50% 정도 더 길었음. PDF 본문 `긁다 @!ai`, `읊다 !4i` 예시를 직접 디코딩해야 발견 가능했던 누락.

### 8. 제16항 성/정/청 특례 미구현 (`250f863`)
**ㅅ/ㅆ/ㅈ/ㅉ/ㅊ + ㅓ + ㅇ받침** 음절은 영 약자 (⠻) 를 적용해야 함. 반대로 셩/졍/쳥 (같은 초성 + ㅕ + ㅇ) 은 풀어쓰기. 이 규칙은 음운이 아니라 점자 표기상 약속이므로 직관적으로 발견 불가능 — **PDF 제16항 직독** 후에야 구현 가능. 성가·정성·청년 같은 흔한 단어들이 PDF 규정과 어긋난 채로 출력되어왔음.

### 9. 한글 문장 부호가 UEB식 (`757f780`)
쉼표 `⠂`, 쌍점 `⠒`, 소괄호 `⠷⠾`, 별표 `⠡⠔` 등은 **통일 영어 점자 (UEB) 규격** — 한국 점자 규정 제32항 [붙임] 의 "**한글 본문 부호 vs 로마자 사이 부호**" 구분을 무시했었음. PDF 비교표 직독 후 `KOREAN_PUNCT` / `UEB_PUNCT` 분기 + 컨텍스트 추적 (`prev_alpha_kind`) 구현. 한글 본문에서 `,` 는 `⠐`, `(가)` 는 `⠦⠄⠫⠠⠴` 로 정정.

### 교훈

- **1차 출처 (국립국어원 PDF) > 2차 도구 (변환기)**. PDF 가 obfuscated 되어 있어도 받아서 직접 디코딩해야 함. 6·7·8·9 번 버그 모두 PDF 직독 단 하루만에 발견. **t.hi098123 같은 변환기는 보조 sanity check 일 뿐**.
- 한국 점자 약자 = **단독 11자 + VF 14종 (을 포함) + 제16항 특례 + 다음절 약어 7종**. 13 종이라 적으면 틀린 것.
- 한국 점자 문장 부호는 **컨텍스트 의존**. UEB 점형 하나로 통일하면 한글 부분이 전부 부정확해짐.
- 메시 검증: `is_watertight=True` 만 보지 말고 **volume 부호** 와 winding 일관성도 확인.
- `replace_all=True` 는 변경 후 반드시 **두 callsite 모두** 결과 확인.

이 프로젝트의 점자 매핑은 그래서 AI 단독 결과물이 아니라 **사용자가 실제 부착하고 손으로 만져보며 잡아 준 검증된 매핑** 입니다. 감사합니다 🙇
